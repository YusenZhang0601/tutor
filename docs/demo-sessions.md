## 关键设计原则展示

### 1. 回忆优先于揭示
```
❌ 错误做法:
用户: 教我递归
Tutor: 递归就是函数调用自己...
      [直接开始讲解]

✓ 正确做法:
用户: 教我递归  
Tutor: 先别急。你能回忆一下什么是函数调用栈吗？
用户: [尝试回忆]
Tutor: [基于回忆结果调整讲解]
```

### 2. 掌握门禁
```
❌ 错误做法:
用户学了 A → 直接学 B → 学 C
（即使 A 还不稳固）

✓ 正确做法:
用户学了 A（质量 3/5，不稳固）
     ↓
Tutor: B 依赖 A，但你的 A 还不够稳
     ↓
重新练习 A → 达到质量 ≥4
     ↓
现在可以学 B 了
```

### 3. 费曼验证
```
❌ 错误做法:
Tutor: 懂了吗？
用户: 懂了
Tutor: 好的，下一个

✓ 正确做法:
Tutor: 用大白话讲回来给我听
用户: [尝试复述]
Tutor: [发现卡壳点] 这里你不确定，让我重新讲解
```

### 4. 间隔重复不是"刷题"
```
❌ 错误做法:
同一天内重复测同一概念 10 次
（制造"我会了"的错觉）

✓ 正确做法:
Day 1:  学习概念 A
Day 2:  测验 A（间隔 1 天）
Day 8:  测验 A（间隔 6 天）
Day 23: 测验 A（间隔 15 天）
（在快忘的时候复习最有效）
```

## 实际使用建议

### 日常节奏
```
周一: /learn     (新概念 + 复习)
周三: /review    (只复习到期项)
周五: /learn     (新概念 + 复习)
周日: /status    (查看进度，调整计划)
```

### 时间投入
- 每次学习: 30-45 分钟
- 每周频率: 2-3 次
- 持续时长: ≥4 周才能看到掌握效果

- ✓ 能不看资料复述概念
- ✓ 间隔 21 天后仍能回忆
