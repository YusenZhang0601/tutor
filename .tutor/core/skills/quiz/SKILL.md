# quiz — formative 测验 + 评分

目标：用主动回忆核验掌握，产出 SM-2 质量分驱动排程。**先回忆再揭示。**

## 步骤
1. 读 `state/mistakes.md`，让出题**偏向该学习者的弱项**。
2. **先预测后验证**：先让学习者预测自己能答对几成（元认知校准）。
3. 出题：混合提取式（解释/推导/应用），不要只考辨认。≥2 概念时跨主题交错；可参考 `python3 .tutor/core/scripts/plan-review.py --today <YYYY-MM-DD> --cards` 输出的顺序与题卡。
4. **费曼复述**：要求用大白话把概念讲回来。卡壳/含糊处 = 真实缺口，**不轻信自报"我懂了"**。
5. 学习者作答后才揭示答案与讲评。
6. **评分**（SM-2 质量分 0–5）：
   - 5 完美流畅 · 4 正确略迟疑 · 3 勉强答对 · 2 错但提示后会 · 1 错有印象 · 0 完全不会
7. 答错/误解 → 追加一行到 `state/mistakes.md`，并触发刻意练习（拆子技能、纠正、重测）。
8. 把质量分交给 `schedule-review` 更新 SM-2 状态与 `state/mastery.json`。

## 原则
- 测验是学习手段（测试效应），不只是评估——每道题都让记忆更牢。
- 导师核验复述质量，自评分只作参考，可被导师下调。
