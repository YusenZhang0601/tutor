# diagnose — 摸底定起点 + 规划路径

目标：避免"凭空给通用大纲"。先测出真实起点，再排出针对个人的概念序列。

## 步骤
1. 读 profile 了解该学科的目标深度与已有基础。
2. **诊断式提问**：从该学科的关键概念出发，由浅入深抛几个探查题，看学习者：
   - 哪些已会（可跳过或仅复习）
   - 哪些有误解（需重点纠）
   - 从哪个概念开始是"难度贴边"
3. 把学科拆成**原子概念**，标出彼此的**前置依赖**（prerequisites），排成一条可放行的学习路径。
4. 创建学科结构：
   - `subjects/NN-<学科>/INDEX.md`（学科 MOC，含概念路径表）
   - `subjects/NN-<学科>/progress.md`（进度摘要，初始为空）
   - `subjects/NN-<学科>/topics/INDEX.md`
   - 在 `subjects/INDEX.md` 增一行
5. 不要一次创建所有概念笔记——概念笔记在 `teach` 首次教到时才生成。

## 原则
- 诊断题要能区分"真懂"和"背过"，多问"为什么"和"如果……会怎样"。
- 路径要尊重前置：地基不稳不往上盖。
